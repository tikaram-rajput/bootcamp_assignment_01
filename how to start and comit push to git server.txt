## Git Workflow Guide (Tikaram Rajput – Project Setup)

This guide explains how I push my local VS Code project to GitHub step-by-step.

---

### 1. Initialize Repository (First Time Only)

```bash
git init
```

---

### 2. Check Project Status

```bash
git status
```

This helps me see:

* New files (untracked)
* Modified files

---

### 3. Add All Files

```bash
git add .
```

This stages all project files for commit.

---

### 4. Commit Changes

```bash
git commit -m "Initial setup: project structure, FastAPI app, parser, requirements"
```

I always write clear and meaningful commit messages.

---

### 5. Configure Git Identity (Done Once)

```bash
git config --global user.name "tikaram-rajput"
git config --global user.email "tikaramsinghrajput@gmail.com"
```

---

### 6. Connect to GitHub Repository

```bash
git remote add origin https://github.com/tikaram-rajput/multimodal-rag-engine-system.git
```

---

### 7. Push Code to GitHub (First Push)

```bash
git branch -M main
git push -u origin main
```

---

### 8. Daily Workflow

Whenever I make changes:

```bash
git add .
git commit -m "Added ingestion module"
git push
```

---

### 9. Useful Commands

Check configuration:

```bash
git config --global --list
```

Check remote:

```bash
git remote -v
```

Check branches:

```bash
git branch
```

---

### 10. Important Notes

* I do not upload `.env` files (kept in `.gitignore`)
* I commit changes step-by-step instead of one big commit
* I use meaningful commit messages for better tracking
* I ensure the repository is public for assignment submission

---

This workflow helps maintain a clean and professional GitHub repository for my Multimodal RAG project.
